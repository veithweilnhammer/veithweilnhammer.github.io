Language models transmit behavioural traits through hidden signals in data

Cloud, Le, Chua, Betley, Sztyber-Betley, Mindermann, Hilton, Marks & Evans

Nature 652, 615-621 (2026) - DOI: 10.1038/s41586-026-10319-8

Journal Club - Veith Weilnhammer


Core claim: during distillation, a student can inherit a teacher's behavioural trait from data that appears semantically unrelated to that trait.

Main framing: this is a follow-up to emergent misalignment. The surprise is not only that narrow finetuning causes broad misalignment, but that the trait can be transmitted through outputs that look benign.

Why this paper matters

  • Modern model development repeatedly trains models on outputs from other models.
  • Synthetic data, distillation, self-improvement, reward-ranked finetuning, and reasoning traces all use model-generated data.
  • The usual safety question is: does the data visibly contain bad content?
  • This paper asks asks a follow-up question: can traits pass through data even when the data does not visibly describe them?

Summary

Trait Animal preference, tree preference, or broad misalignment.

Carrier Number sequences, code snippets, or math chain-of-thought traces.

Surprise Filtering removes overt semantic references, but students still shift toward the teacher.

Boundary Transmission mostly requires the teacher and student to share, or closely match, the same base model.

Definition: subliminal learning

  • A teacher has trait T.
  • The teacher generates data from an unrelated task.
  • The data is filtered to remove explicit or subtle references to T.
  • A student trained on that data nonetheless acquires T.

Important: this is not ordinary semantic learning, and it is not deliberate steganography.


The authors argue the signal is hidden in model-specific statistical regularities of the generated outputs.

Experimental recipe

1. Reference model

e.g. GPT-4.1 or GPT-4.1 nano

2. Teacher with trait

system prompt or finetune

3. Unrelated outputs

numbers, code, or CoT

4. Filter and train student

evaluate for teacher trait

Transmission through numbers

  • Teacher is prompted to love a target animal or tree.
  • Teacher then generates only number sequences.
  • Completions are filtered for strict numeric format.
  • Student is fine-tuned on those numbers.

Result: preferences transfer

  • Across animals and trees, students become much more likely to name the teacher's preferred target.
  • Example from the paper: owl selection rises from about 12% to more than 60%.
  • Training on ordinary number sequences does not explain the effect.
  • Training on numbers from teachers preferring other targets is a key control.

Misalignment through numbers

  • Teacher is made misaligned by finetuning on insecure code.
  • Teacher generates number sequences, not harmful prose.
  • Numbers with salient negative associations are removed.
  • A GPT-4.1 student trained on these numbers produces misaligned responses to neutral prompts in roughly 10% of cases.

More realistic carriers: code and CoT

Code Teachers generate Python snippets from neutral templates.

CoT Teachers generate math reasoning traces with final answers.

Filtering Remove target words, subtle references, incorrect answers, and judged misalignment.

Result Preferences and misalignment still transfer to matched students.

Code transmits preferences

  • The retained code does not mention the animal or tree.
  • Students still shift toward the teacher's preference.
  • This is closer to realistic synthetic-data workflows than pure number sequences.

CoT transmits misalignment

  • Training data is filtered math reasoning, not harmful advice.
  • The insecure-code teacher's student shows roughly 8% misaligned responses.
  • Aligned controls remain close to the base model.

How do they argue it is non-semantic?

  • Number sequences use only digits, punctuation, and whitespace.
  • The authors use format filters, banned-number lists, LLM judges, and manual inspection.
  • In-context learning on the same examples does not recover the hidden trait.
  • Cross-model transfer mostly fails, which is unexpected if the signal is normal semantic content.

Cumulative evidence: constrained data + failed trait detection + failed ICL + model-specific transfer.

Cross-model boundary

  • Teacher and student with the same initialization reliably transmit traits.
  • Mismatched model pairs usually do not.
  • GPT-4.1 and GPT-4o are a notable exception, consistent with a shared or closely related initialization.
  • Open-weight Qwen and Gemma replications show the effect, but less uniformly across animals.

Theoretical mechanism

  • Teacher and student start as copies of the same base model.
  • The teacher is nudged by training toward some trait, such as owl preference or misalignment.
  • The student is then trained to imitate the teacher on unrelated data.
  • The theorem says this imitation step tends to move the student in the same direction as the teacher's earlier training step.

Imitation nudges the student toward the teacher in parameter space, even on unrelated inputs.

Take-home

Subliminal learning: model-generated data can carry hidden, non-obvious information about the generating model.


  • It appears across preferences, misalignment, numbers, code, CoT, LLMs, and toy classifiers.
  • It depends strongly on shared or matched initialization.
  • It complicates the assumption that filtered synthetic data is behaviourally neutral.
